# 可做性与风险 v2

## 可做性判断

当前判断：**可做，但需要把问题重新定位为 self-localized but neighbor-unknown 的 ISAC+MARL 邻居发现。**

最稳妥的论文定位不是 “ISAC-assisted beam alignment”，而是：

> ISAC-assisted and MARL-enabled topology-aware narrow-beam neighbor discovery for self-localized fully distributed UAV swarms.

即：UAV 自身定位姿态可用，用于建立坐标基准和自身运动上下文；ISAC 输出不完美候选波位先验；链路层协议和 MARL 策略负责 mode/beam 调度、双向握手确认和有限时间拓扑质量优化。

## 支持可做性的理由

1. 自定位不等于邻居发现。节点知道自己在哪里，不意味着知道邻居是谁、在哪个方向、是否可通信、当前是否接收。
2. ISAC 能提供目标感知和空波位排除能力，弥补自定位无法感知环境候选目标的问题。
3. 传统 DND 文献主要优化扫描序列，缺少 ISAC prior 和自定位辅助的可学习策略。
4. 现有 ISAC beam management 多为 BS/gNB/AP 主导，不是 fully distributed U2U。
5. MARL 可学习复杂 mode/beam 调度，但必须通过参数共享和局部固定维观测解决规模迁移。

## 主要风险与应对

| 风险 | 严重度 | 说明 | 应对 |
|---|---|---|---|
| 与 ISAC beam rendezvous 重叠 | High | Airborne + ISAC + beam rendezvous 题面接近。 | 精读边界；突出 neighbor discovery、身份未知、MARL 规模迁移和拓扑优先。 |
| 与 3D UAV DND 重叠 | High | SkyOrbs 已做快速 3D UAV ND。 | 作为强基线；突出 ISAC prior、自定位辅助 MARL 和 topology-aware objective。 |
| 自定位削弱 ISAC 必要性 | Medium | 审稿人可能问：既然 UAV 有位置，为什么还需要 ISAC？ | 回答：自定位只知道自身状态，不知道邻居状态、链路可达性、对方波束和空波位。 |
| MARL 规模泛化失败 | High | 小规模训练策略可能在大规模中因碰撞、密度和观测分布变化失效。 | 参数共享、局部固定维观测、GNN/attention pooling、mean-field、domain randomization 和大规模直接迁移测试。 |
| MARL 黑箱不可解释 | Medium | TWC/TCOM 可能要求机制解释。 | 保留规则 I-TAP-ND 作为可解释基线，MARL 作为规则-数据双驱动增强。 |
| 性能提升来自 oracle 信息泄露 | High | 仿真若直接用未发现邻居真实位置会被否定。 | 明确区分 self state、ISAC observation、protocol state、simulator truth。 |
| 拓扑价值无法在发现前精确估计 | Medium | 发现前不知道全局拓扑。 | 使用方向覆盖、候选密度、已发现局部图和自运动上下文作为代理。 |

## Go / No-Go 条件

### Go

- 文献确认没有工作同时覆盖 fully distributed U2U + ISAC prior + self-localized neighbor-unknown model + transferable MARL + topology-aware finite-time discovery。
- Beam-cell 级最小仿真显示 ISAC prior 在中等误差下仍降低空扫和尾部时延。
- MARL-I-TAP-ND 至少在部分测试规模上优于规则 I-TAP-ND 或能在大规模保持接近规则方法的性能。
- 小规模训练到大规模部署的性能保持率可量化。

### No-Go 或收缩范围

- 若 ISAC 误差稍大即失效，则主贡献转向 uncertainty-aware robust protocol。
- 若 topology-aware 无稳定收益，则拓扑指标降为次要实验。
- 若 MARL 不能迁移，则保留规则 I-TAP-ND 为主方法，MARL 作为负结果或后续工作。
