# 可做性与风险 v1

## 可做性判断

当前判断：**可做，但必须收窄并强化问题定义。**

最稳妥的论文定位不是“ISAC-assisted beam alignment”，而是：

> ISAC-assisted topology-aware narrow-beam neighbor discovery for fully distributed UAV swarms.

即：ISAC 只作为物理层感知能力，输出不完美的候选波位先验；链路层协议负责概率扫描、双向握手确认和有限时间拓扑质量优化。

## 支持可做性的理由

1. 传统 DND 文献主要优化随机/确定性扫描序列，缺少对准前 ISAC 感知先验。
2. UAV/airborne DND 文献已覆盖无先验、异步、3D 或高动态，但仍以盲扫或规则扫描为主。
3. ISAC beam management 文献证明感知先验可减少 beam search，但多为 BS/gNB/AP 主导，不是 fully distributed U2U。
4. 现有工作很少把邻居发现过程与有限时间拓扑质量、一致性收敛直接耦合。
5. 可以先用 beam-cell 级抽象仿真验证，不需要一开始实现复杂物理层波形。

## 主要风险与应对

| 风险 | 严重度 | 说明 | 应对 |
|---|---|---|---|
| 与 ISAC beam rendezvous 重叠 | High | Hong 2024 已做 airborne + ISAC + beam rendezvous。 | 精读该文，明确其是否假设已知通信对象；突出 neighbor discovery、身份未知和拓扑形成。 |
| 与 3D UAV DND 重叠 | High | SkyOrbs 2024 已做快速 3D UAV DND。 | 将其作为强基线；突出 ISAC prior 和 topology-aware priority。 |
| 感知先验不是新概念 | High | CommRad 和 adaptive beam probing 已用 sensing prior 缩小 beam search。 | 强调 fully distributed U2U、双向 discovery、误感知鲁棒和拓扑收益。 |
| 物理层抽象过粗 | Medium-High | TWC/TCOM 可能要求更具体的 ISAC 模型。 | 建立最小物理层抽象：角度误差、虚警、漏检、置信度、感知周期与 sensing overhead。 |
| 拓扑价值无法在发现前估计 | Medium | 对准前不知道邻居身份和全局拓扑。 | 使用本地代理指标：方向覆盖、已发现度数、空间稀缺性、历史发现分布，而不是假设全局拓扑。 |
| 性能提升来自 oracle 信息泄露 | High | 仿真若直接用真实邻居位置会被否定。 | 明确区分 true topology、ISAC observation 和 protocol state；设置 oracle 只作为上界。 |
| RL/智能方法收益有限 | Medium | 用户已有经验表明三维空间下 RL 提升有限。 | 不把 RL 作为主贡献；主机制是搜索空间压缩和拓扑优先。 |

## Go / No-Go 条件

### Go

- 文献确认没有工作同时覆盖 fully distributed U2U + ISAC prior + pre-alignment ND + topology-aware finite-time discovery。
- Beam-cell 级最小仿真显示在中等虚警/漏检下仍能降低空扫和尾部时延。
- 拓扑优先机制相比 ISAC-only 版本能提升连通性或一致性收敛速度。

### No-Go 或收缩范围

- 若 Hong 2024 已完整覆盖 ISAC-assisted distributed neighbor discovery，则需要转向 topology-aware robust discovery。
- 若 ISAC 误差稍大即失效，则主贡献应转为 uncertainty-aware robust protocol，而不是空波位剪枝。
- 若拓扑优先无法带来稳定收益，则论文应只保留 ISAC-assisted ND，并把拓扑指标作为次要实验。
