# 创新点假设 v1

## C1. ISAC 辅助 beam-cell occupancy prior

将 ISAC 感知结果抽象为三维 beam-cell occupancy prior，而不是直接作为邻居位置或最佳通信波束。该 prior 包含：

- 候选波位集合；
- 波位占据概率；
- 置信度；
- 虚警率；
- 漏检率；
- 角度误差；
- 信息更新时间。

预期贡献：把传统 DND 的均匀扫描空间压缩为带置信度的候选搜索空间。

## C2. 不确定性感知的非均匀随机邻居发现

在完全分布式条件下，节点仍然独立随机选择工作模式和波束，但随机分布由 ISAC prior 和探索概率共同决定：

```text
P(select beam b) = f(occupancy_confidence, uncertainty, exploration_floor)
```

预期贡献：保留随机方法低平均时延和灵活性，同时缓解均匀盲扫的空扫和长尾问题。

## C3. 拓扑质量优先的候选波位排序

在有限发现时间内，不只追求任意邻居发现，而是对候选波位赋予 topology-aware priority：

```text
priority(b) = occupancy_prior(b) * link_quality_proxy(b) * topology_value_proxy(b) / scan_cost(b)
```

可选拓扑代理：

- 本地度数不足；
- 空间方向覆盖稀缺性；
- 可能桥接连通分量的方向；
- 对代数连通度的局部近似贡献；
- 对一致性收敛速度的预期收益。

预期贡献：把邻居发现和 swarm 协同拓扑形成关联起来。

## C4. 误感知鲁棒发现机制

ISAC prior 不作为最终发现结果；邻居身份和链路存在必须通过链路层握手确认。

机制包括：

- 低置信波位保底探索；
- 置信度随时间衰减；
- 感知候选与握手结果相互校正；
- 虚警惩罚和漏检恢复；
- sensing / Tx / Rx 比例自适应。

预期贡献：回应 multipath 下“seeing is not always believing”的审稿风险。

## C5. 与现有工作的边界

| 现有方向 | 常见贡献 | 我们的边界 |
|---|---|---|
| ISAC beam tracking | 利用感知预测已知链路波束 | 我们研究对准前未知邻居发现 |
| Sensing-assisted IA | BS/gNB/AP 主导初始接入 | 我们研究完全分布式 U2U |
| 3D UAV DND | 优化三维扫描序列或机械路径 | 我们引入 ISAC prior 排除空波位 |
| RL-based DND | 从历史发现反馈学习策略 | 我们使用对准前感知先验，并显式建模误感知 |
| Topology control | 发现后优化拓扑 | 我们在发现阶段优先建立关键链路 |
