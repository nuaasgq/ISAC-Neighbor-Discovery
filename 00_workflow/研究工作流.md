# 通感一体化辅助的窄波束邻居发现研究工作流

## 0. 项目定位

### 研究主题

面向完全分布式、无中心控制、对准前未知邻居状态的无人机集群网络，研究通感一体化（ISAC）能力如何作为链路层邻居发现协议的物理层先验，减少三维窄波束邻居发现中的空波位扫描，并在有限时间资源下优先建立支撑拓扑质量提升的关键链路。

### 初始研究边界

- 网络形态：UAV-UAV 自组织集群网络，不考虑 UAV-BS 蜂窝接入作为主场景。
- 协议假设：完全分布式，无中心调度；UAV 可知道自身位置、速度、航向和姿态；对准前未知邻居身份、邻居实时位置、邻居波束状态、历史链路和全局拓扑。
- 波束条件：三维窄波束扫描，方位角和俯仰角共同决定波位。
- ISAC 定位：不主攻波形设计和物理层最优波束赋形，而是把 ISAC 抽象为链路层可调用的感知能力。
- 论文定位：ISAC + 网络跨层论文，重点是邻居发现协议和拓扑形成过程。
- 目标期刊：优先考虑 IEEE TWC / IEEE TCOM 风格，需要协议机制、系统模型、理论分析或强仿真验证同时成立。

### 核心问题 v0

在无中心、节点自身定位姿态可用但对准前未知邻居状态的三维窄波束 UAV 集群网络中，ISAC 感知得到的候选波位占据信息能否作为链路层邻居发现先验，使随机或学习式邻居发现从均匀盲扫转变为感知引导的概率搜索，并在降低平均/尾部发现时延的同时提升有限时间内的拓扑连通质量？

### P4. 协议与算法设计

#### 主算法方向

暂定名称：

`ISAC-Assisted Topology-Aware Probabilistic Neighbor Discovery`

核心思想：

1. 节点周期性用通信波形进行感知，得到本地三维波位占据置信图。
2. 节点根据置信图构建非均匀扫描概率，而不是在所有波位上均匀随机扫描。
3. 协议保留探索概率，避免 ISAC 漏检导致邻居永久不可发现。
4. 对候选波位引入拓扑收益权重，有限时间内优先发现更可能提升连通性的链路。
5. 邻居一旦完成握手，再进入正常链路维护或 beam tracking，不混淆 discovery 和 tracking。

#### 候选机制

- 感知辅助波位剪枝：过滤低概率空波位。
- 置信度驱动随机扫描：高概率波位获得更大 Tx/Rx 选择概率。
- 拓扑质量优先：对空间方向稀缺、节点度不足或可能桥接分量的波位加权。
- 鲁棒探索：固定小概率扫描低置信波位。
- 感知一致性校验：感知目标必须通过链路层握手确认，避免把虚警当邻居。

#### 产物

- `04_protocol/protocol_design.md`
- `04_protocol/pseudocode.md`

- `04_protocol/theoretical_properties.md`

- 能和随机、确定性、RL、无拓扑权重版本公平比较。

### P5. 仿真实现

#### 最小可用仿真

第一版不追求全物理层精细建模，先实现离散波位级仿真：

- 三维空间 UAV 随机部署或编队部署。
- 球面波位离散化。
- 每个时隙节点选择 sensing / Tx / Rx。
- ISAC 模块输出候选波位和置信度。
- 协议模块选择波束和工作模式。
- 链路层握手判定是否发现邻居。
- 输出发现时延和拓扑质量指标。

#### 基线算法

- Uniform Random ND：均匀随机扫描。
- Deterministic Scan ND：固定序列扫描。
- Topology-Blind ISAC ND：只用 ISAC 剪枝，不考虑拓扑。
- RL-based ND：如果已有强化学习代码，可作为智能基线。
- Oracle-assisted ND：理想感知上界，仅用于性能边界。

#### 产物

- `05_simulation/src/`
- `05_simulation/configs/`
- `05_simulation/results_raw/`
- `05_simulation/run_experiments.md`

## 4. 决策检查点

### Checkpoint A：方向可做性

进入下一阶段前必须回答：

- 和 ISAC beam tracking 的根本区别是什么？
- 和 airborne beam rendezvous 的根本区别是什么？
- 和传统 directional neighbor discovery 的根本区别是什么？
- ISAC 感知错误是否会使方法退化到不可用？

### Checkpoint B：模型一致性

进入算法实现前必须确认：

- 没有使用对准前不可获得的信息。
- ISAC 抽象能力不等价于提前知道邻居位置。
- 节点决策是本地的，不依赖中心控制。
- 拓扑质量指标可由已发现链路或本地近似信息计算。

### Checkpoint C：实验可信度

进入论文写作前必须确认：

- 所有基线公平使用相同波束、时隙、节点运动和信道条件。
- 所有性能提升不是由 oracle 信息泄露造成。
- 所有图表可复现。
- 消融实验能解释收益来源。

### Checkpoint D：投稿准备

投稿前必须确认：

- 文献引用真实、准确、无伪造。
- 贡献表述不过度声称。
- TWC/TCOM 的物理层深度和网络协议深度达到平衡。
- 代码、参数、随机种子和数据处理流程可追溯。
