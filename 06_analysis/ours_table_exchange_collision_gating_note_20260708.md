# 我方表交换、冲突控制与多 RF 门控说明

## 表交换

当前新增协议为 `improved_rl_isac_tables`。它保留 `improved_rl_isac` 的 ISAC 信念引导与拓扑感知波束选择，但在两个 UAV 成功握手后，双向 piggyback 交换两类表项：

- 邻居信息表：已成功发现的邻居节点或链路信息。接收方根据自身位置和表中目标位置，计算指向该潜在邻居的本地波束，并提升该波束的候选优先级。
- 邻居感知表：近期高置信度的 ISAC 感知阳性波束。接收方把对方感知到的潜在目标映射为自己的候选波束，降低从零盲扫的概率。

该机制仍是完全分布式的：只有已经完成握手的两个节点交换局部表项，没有中心控制器，也不要求对准前知道其他节点的 GNSS/惯导/历史轨迹。

## 冲突控制

窄波束邻居发现不是“选中目标波束就一定成功”。一个接收节点在同一时隙可能被多个发送节点命中，此时会发生握手冲突，导致所有竞争发送都失败。随着节点数、候选波束集中度、RF 链数增加，冲突会成为主要瓶颈。

冲突控制就是让智能体根据局部统计动态调节接入行为，例如：

- 某候选波束近期碰撞失败多，则降低继续发送概率，增加接收或等待概率。
- 某节点已达到目标度数，则减少主动发送，给低度节点让出发现机会。
- 某波束被多次感知为热门方向，则不让所有节点同时冲向该波束，而是做随机退避或分时尝试。

在 MARL 中，它可以体现在状态、动作和奖励三处：状态加入 `collision_fail_count`、候选波束拥塞度、节点度数缺口；动作加入 Tx/Rx/idle 或退避选择；奖励对碰撞失败和重复竞争施加惩罚。

## 多 RF 激活门控

多 RF 链意味着一个节点可以在同一时隙激活多个波束。但实验已经显示，多 RF 不一定单调提升发现率：随机盲扫会受益于更多并行扫描，而有 ISAC 引导的方法如果同时激活过多热门候选波束，可能引发更高冲突和重复扫描。

多 RF 激活门控就是不把所有 RF 链默认开满，而是让智能体决定：

- 本时隙激活几个 RF 链：0, 1, ..., K。
- 哪些候选波束值得并行激活。
- 是否只保留一个高置信主波束，把其他 RF 链留给探索或监听。
- 当碰撞压力高时，主动减少 Tx 波束数或切换为 Rx。

因此，多 RF 门控不是物理层波束形成算法，而是数据链路层的资源调度策略。它适合作为我方网络结构创新的一部分：网络可以输出主波束、辅助波束 mask、RF 激活数和 Tx/Rx 角色，由规则层保证协议约束，由学习层适配不同节点规模和波束宽度。

## 下一步实现建议

`improved_rl_isac_tables` 只是把表交换接入现有规则策略。下一步应实现一个新的 MARL 协议，例如 `marl_isac_table_gated`，把表交换、冲突控制和 RF 门控统一放进可学习策略：

- observation：本地 ISAC belief、age、success/fail/collision 统计、邻居表摘要、感知表摘要、节点度数缺口。
- action：mode、主波束、辅助波束 mask、RF 激活数、表项融合权重或置信门控。
- reward：发现新链路奖励、拓扑连通度奖励、空扫惩罚、碰撞惩罚、能耗惩罚、过度集中扫描惩罚。

这样才能从“ISAC 辅助规则协议”升级为“ISAC+表交换+冲突感知+多 RF 门控 MARL 协议”。
