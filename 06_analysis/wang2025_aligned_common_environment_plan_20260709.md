# Wang2025-Aligned Common-Environment Experiment Contract

## Purpose

This file records the fairness rule for comparing MARL against Wang-style
neighbor discovery in the same experimental environment.

The main comparison must answer only one question:

> Given the same physical/link-layer environment and the same table-exchange
> mechanism, which decentralized action decision policy discovers neighbors
> faster?

## Fixed Common Environment

Main comparison rows must use the same simulator environment protocol:

- `common_env_protocol = wang2025_isac_tables`
- single RF chain
- Wang-style beam grid from `05_simulation/configs/wang2025_reproduction_smoke.yaml`
- TX-coupled piggyback ISAC only
- no standalone SENSE action in the fair-comparison action set
- collision handling: if multiple TX nodes hit the same RX in the same slot,
  no edge is discovered
- duplicate-edge no-reply: an already discovered peer does not trigger another
  ACK/table exchange
- neighbor table exchange after successful first interaction
- sensing table exchange after successful first interaction
- identical mobility, range, slot duration, sensing-error model, and episode
  horizon across all rows

## What May Differ

Only the decentralized action decision policy may differ. In the main table,
each method decides:

- mode: TX, RX, or IDLE
- communication beam index

No method should receive an extra hidden execution rule that another method
does not receive.

## Main Rows

The strict main rows generated by
`06_analysis/scripts/run_wang2025_aligned_marl_matrix.py` are:

- `uniform_trx_idle_random`: random TX/RX/IDLE and random beam action policy
- `wang2025_isac_tables`: Wang-style sensing-table action policy, executed
  inside the same common environment
- `budgeted_collision_aware_isac`: rule-based ISAC action policy, executed
  inside the same common environment
- `marl_wang_isac_tables_discovery_first`: MARL policy trained and evaluated
  inside the same common environment

The MARL main specification uses the non-gated `contention_shared` network so
the final sampled action is the executed action. This avoids a separate
environment-side access-gate rewrite in the main fair comparison.

## Not Main Fair-Comparison Rows

The following are still useful but should be treated as ablations, not as the
main Wang-vs-MARL comparison:

- Wang no-collaboration / communication-table / sensing-table variants when
  each variant changes environment capability rather than only action policy.
- MARL without ISAC, because it removes a physical-layer capability from the
  environment/observation contract.
- MARL with access-gate execution rewriting, unless every compared method is
  given the same explicit action-space and execution semantics.
- Any result where `sense_actions > 0` in the main fair-comparison rows.

```powershell

```


## Acceptance Checks

Before using a table in the manuscript, check:

- `manifest.json` has `common_env_protocol = wang2025_isac_tables`.
- `manifest.json` states the main action space is TX/RX/IDLE plus beam.
- `aggregate_metrics.csv` has no main row with `sense_actions_mean > 0`.
- `per_episode_summary.csv` includes `env_protocol` and
  `action_policy_protocol` for baseline rows.
- MARL training manifest records `network = contention_shared` for the strict
  main fair-comparison run.
