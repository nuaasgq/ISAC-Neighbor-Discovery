# 系统模型假设审查 v1

## 目的

本文件记录新设定下的系统模型压力测试，核心边界是：

> 自身真值可用，未发现邻居真值不可用，真实全局拓扑不可用。

## 修订后的关键假设

| 假设 | 建议表述 | 风险 |
|---|---|---:|
| 自身定位姿态可用 | UAV 可使用自身位置、速度、航向、姿态、俯仰和横滚等自状态信息 | 中 |
| 未发现邻居状态未知 | 对准前不知道未发现邻居身份、实时位置、姿态、速度、波束状态和历史链路 | 高 |
| 本地波束码本 | 每个 UAV 知道自身天线阵列坐标系下的 beam cell，并可由自身姿态映射到 global-frame 方向锥 | 中 |
| ISAC prior | ISAC 输出 beam-cell occupancy prior、置信度、粗距离/多普勒等候选目标证据，不确认邻居 | 高 |
| 完全分布式 | 没有中心调度、全局拓扑、全局 Tx/Rx 配对 | 高 |
| topology-aware | 发现前只能使用自身状态、ISAC 候选密度、方向覆盖、已发现邻居摘要等代理 | 高 |

## 主要质疑与回答

| 审稿人质疑点 | 回答 |
|---|---|
| 既然 UAV 有定位，为什么还需要邻居发现？ | 自定位不等于邻居发现。节点知道自己在哪里，不知道其他节点是谁、在哪里、是否可通信、是否正在接收、波束朝向哪里。邻居关系必须通过窄波束链路层握手确认。 |
| 自定位是否削弱 ISAC 必要性？ | 不削弱。自定位提供坐标基准，ISAC 提供环境中候选目标和空波位证据。 |
| ISAC prior 是否等价于知道邻居位置？ | 否。prior 是概率场，不含 ID、不保证可通信、不保证 LoS、不保证对方监听。 |
| MARL observation 是否会偷用全局真值？ | Actor 执行期只能使用自身状态、ISAC prior、已发现邻居摘要和本地历史；critic 训练期可用仿真全局信息。 |
| topology-aware 是否使用未知邻居真值？ | 不允许。发现前只用方向覆盖、候选密度、度数缺口、自身运动上下文等代理。 |

## 必须写清的约束

- 协议可访问自身位置、速度和姿态。
- 协议不可访问未发现邻居 ID、实时位置、速度、姿态和波束状态。
- 协议不可访问真实全局拓扑和全局代数连通度。
- ISAC prior 只能影响波束选择概率，不能直接确认邻居。
- 邻居关系必须由 beacon/ACK 或等价链路层握手确认。
- 仿真器可以知道真实几何用于判定成功，但 actor 和规则协议不能使用这些真值。
