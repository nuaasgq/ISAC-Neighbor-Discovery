# 系统模型假设审查 v0

## 目的

本文件记录 Sprint 2 中对系统模型的审稿人视角压力测试，避免后续协议和仿真偷用对准前不可获得的信息。

## 高风险假设

| 假设 | 建议表述 | 风险 |
|---|---|---:|
| 无 GNSS/惯导/历史邻居状态 | 对准前节点不知道自身全局位置、姿态、速度，也不知道邻居 ID、距离、方位、历史链路 | 高 |
| 本地波束码本 | 每个 UAV 只知道自身天线阵列坐标系下的离散 beam-cell index | 中 |
| ISAC prior | ISAC 只输出 beam-cell occupancy prior，不输出邻居身份，不保证目标可通信 | 高 |
| 完全分布式 | 没有中心调度、全局拓扑、全局同步 Tx/Rx 配对 | 高 |
| 邻居确认 | 只有 Tx/Rx 波束互相覆盖并完成 beacon/ACK，才确认邻居 | 中 |
| topology-aware | 发现前只能使用局部代理，不能使用真实拓扑增益 | 高 |

## 主要质疑与修正

| 审稿人质疑点 | 风险 | 修正 |
|---|---:|---|
| 无 GNSS/惯导时怎么知道方位角和俯仰角？ | 高 | 不使用全局方位；定义 body-frame local beam-cell。 |
| ISAC prior 是否等价于知道邻居位置？ | 高 | prior 是概率场，不含 ID、不保证可通信、不保证 LoS、不保证对方监听。 |
| 感知目标不等于通信邻居。 | 高 | 将 sensed target 分为可通信 UAV、不可通信目标、虚假目标；握手后才入邻居集。 |
| topology-aware 在发现前不可计算。 | 高 | 拆成发现前 proxy priority 和发现后 confirmed topology priority。 |
| Tx/Rx/Sensing 是否偷用同步或中心调度？ | 高 | 使用 slotted-but-unscheduled；节点独立随机选择模式。后续可做异步偏移实验。 |
| Sensing 和通信是否能同时进行？ | 中 | 主模型半双工；同时 ISAC 只作为 optimistic upper bound。 |
| ISAC prior 是否需要历史轨迹？ | 中 | 只由当前或最近 sensing 快照生成，并随时间衰减；不使用邻居轨迹。 |
| 角度误差会让高置信波位错位。 | 高 | 用 beam-cell 混淆核建模，而不是单点标记。 |
| 单边 sensing 不足以完成双向发现。 | 高 | sensing 只影响本节点波束分布；成功仍要求 Tx/Rx 双向覆盖和握手。 |

## 必须写入模型的约束

- 协议不可访问全局位置、全局姿态、全局拓扑。
- 对准前不可访问邻居 ID、历史链路、相对轨迹。
- ISAC prior 只能影响波束选择概率，不能直接确认邻居。
- 所有 Tx/Rx/Sensing 决策由节点本地随机完成。
- topology-aware 在发现前只能是代理指标，不能使用真实拓扑增益。
- 仿真器可以知道真实几何用于判定成功，但协议输入不能使用这些真值。
