# 系统模型 v0

## 1. 网络模型

考虑由 `N` 个无人机构成的 UAV-UAV 自组织集群网络。网络中不存在中心控制器、基站、簇首或全局调度器。每个 UAV 只根据本地状态独立决策。

本研究采用 **self-localized but neighbor-unknown** 设定：每个 UAV 可知道自身位置、速度、航向和姿态，但在完成链路层发现握手前，不知道未发现邻居的身份、实时位置、运动状态、波束状态和全局拓扑。

### 关键约束

- 对准前，节点不知道未发现邻居身份、邻居实时位置、邻居速度、邻居波束状态和邻居历史链路。
- 节点可使用自身定位和姿态信息，包括自身位置、速度、航向、俯仰和横滚。
- 节点可将自身天线阵列坐标系下的本地波位映射到当前全局方向锥，但该映射不包含未发现邻居状态。
- 节点可通过 ISAC 波形获得本地感知观测，但该观测不等价于已知邻居位置。

## 1.1 自身状态模型

节点 `i` 在时隙 `t` 的自身状态为：

```text
s_i^self(t) = {x_i(t), v_i(t), psi_i(t), theta_i(t), phi_i(t), R_i(t), Sigma_i^loc(t)}
```

其中：

- `x_i(t)`：自身三维位置。
- `v_i(t)`：自身速度。
- `psi_i(t), theta_i(t), phi_i(t)`：航向角、俯仰角和横滚角。
- `R_i(t)`：body-frame 到 global-frame 的旋转矩阵。
- `Sigma_i^loc(t)`：自定位误差协方差，可在 MVP 中先设为零或小常数。

自定位提供的是坐标基准，不提供邻居身份或链路可达性。

## 2. 本地波位模型

每个节点 `i` 在自身 body-frame / antenna-frame 下维护三维波束码本：

```text
B_i = {b_i^1, b_i^2, ..., b_i^M}
```

每个 beam cell 对应本地天线坐标系中的一个离散方向单元：

```text
b_i^m = (a_i^m, e_i^m), a_i^m in {1,...,A}, e_i^m in {1,...,E}, M = A * E
```

其中：

- `A` 是本地水平扫描维度的 beam cell 数。
- `E` 是本地垂直扫描维度的 beam cell 数。
- `M` 是节点单轮三维扫描空间大小。

借助 `R_i(t)`，节点可知道每个本地 beam cell 在 global-frame 中的方向锥：

```text
B_i^global(t) = R_i(t) B_i^body
```

该方向锥只表示“我当前朝哪些空间方向扫描”，并不表示该方向存在邻居。仿真器内部可以使用全局几何坐标来判定真实相对方向，但协议输入不能包含未发现邻居的真实位置或身份。

## 3. 时隙与工作模式

时间被离散为时隙 `t = 1, 2, ...`。在每个时隙，节点 `i` 选择一种工作模式：

```text
m_i(t) in {SENSE, TX, RX, IDLE}
```

模式互斥：

- `SENSE`：节点发送/接收 ISAC 感知波形，更新本地 beam-cell occupancy prior。
- `TX`：节点在选定 beam cell 上发送邻居发现 beacon。
- `RX`：节点在选定 beam cell 上监听邻居发现 beacon。
- `IDLE`：节点空闲或执行非发现任务。

第一版 MVP 采用 slotted-but-unscheduled 模型以降低实现复杂度。节点共享粗粒度 slot 边界或本地 slot 时钟，但没有中心调度，也没有预先分配 Tx/Rx 对。异步偏移和漂移作为后续鲁棒性扩展。

## 4. ISAC 感知观测模型

节点 `i` 在时隙 `t` 感知后得到本地观测：

```text
O_i(t) = { (b_i^m, p_i^m(t), u_i^m(t), tau_i^m(t), rhat_i^m(t), vhat_i^m(t)) | b_i^m in B_i }
```

其中：

- `p_i^m(t)`：beam cell `b_i^m` 存在潜在反射目标的 occupancy prior。
- `u_i^m(t)`：不确定性或置信度宽度。
- `tau_i^m(t)`：该观测的时间戳。
- `rhat_i^m(t)`：可选粗距离估计。
- `vhat_i^m(t)`：可选径向速度或 Doppler 粗估计。

`p_i^m(t)` 的含义是：

```text
p_i^m(t) = Pr(b_i^m contains at least one potential target | local sensing observation)
```

它不包含目标 ID，不保证目标是可通信 UAV，不保证对方处于 RX，不保证该方向是最佳通信波束。

自身定位和 ISAC 的关系：

```text
self-localization provides the coordinate reference;
ISAC provides target-aware beam-cell occupancy evidence.
```

ISAC 观测受以下误差影响：

- 虚警：空 beam cell 被误判为存在目标。
- 漏检：存在邻居的 beam cell 未被感知到。
- 角度误差：真实目标所在 beam cell 被映射到相邻 beam cell。
- 非邻居目标：感知到的目标可能不是可通信邻居。
- 多径误导：感知角度可能不是最佳通信方向。

协议只能读取 `O_i(t)`，不能读取真实邻居位置。

感知目标分为三类：

1. 可通信 UAV 邻居。
2. 不可通信目标或非合作目标。
3. 虚假目标。

只有第一类在完成链路层握手后才能成为邻居；ISAC 感知本身不产生邻居边。

## 5. 邻居发现模型

节点 `i` 和 `j` 在时隙 `t` 成功发现彼此，需要满足：

1. 一个节点处于 `TX`，另一个节点处于 `RX`。
2. `TX` 节点选择的本地 beam cell 覆盖 `RX` 节点的真实相对方向。
3. `RX` 节点选择的本地 beam cell 覆盖 `TX` 节点的真实相对方向。
4. 链路预算满足最小接收门限。
5. 该时隙无不可恢复冲突。
6. 完成链路层握手，确认对方身份。

发现成功后，边 `(i, j)` 加入已发现链路集合：

```text
E_D(t) = E_D(t-1) union {(i,j)}
```

真实可通信邻居集合 `E^*` 只用于仿真评估，不作为协议输入。

## 6. 拓扑状态

协议层可用状态：

- 自身位置、速度、航向和姿态。
- 本地 beam cell 到 global-frame 方向锥的映射。
- 已发现邻居集合 `N_i^D(t)`。
- 已发现节点度 `d_i^D(t)`。
- 最近成功发现的 beam cell 分布。
- 本地 beam-cell occupancy prior。
- 本地发现失败/虚警统计。
- 握手后可交换的局部信息，例如已发现度数或链路质量摘要。

协议层不可用状态：

- 全局真实拓扑 `G^*`。
- 未发现邻居的真实身份。
- 未发现邻居的真实位置。
- 未发现邻居的姿态、速度和波束状态。
- 全局代数连通度。

因此，topology-aware priority 必须使用本地代理指标，而不是全局真实拓扑指标。

拓扑感知分为两个阶段：

- 发现前：只能使用 pre-discovery proxy，例如方向覆盖稀缺性、候选目标密度、已发现度数不足和长期未探索方向。
- 发现后：可以使用 confirmed topology state，例如已确认邻居、局部度数、链路质量和可选二跳摘要。

## 7. 信息边界

| 信息 | 协议可用 | 仿真可用 | 说明 |
|---|---|---|---|
| 自身位置/速度/姿态 | 是 | 是 | UAV 自定位能力 |
| 本地 beam cell 到全局方向锥映射 | 是 | 是 | 只依赖自身姿态 |
| 本地 beam cell index | 是 | 是 | 节点本地坐标定义 |
| ISAC occupancy probability | 是 | 是 | 不完美观测 |
| 已发现邻居 ID | 是 | 是 | 仅握手后可用 |
| 握手后局部度数摘要 | 可选 | 是 | 仅确认邻居后交换 |
| 未发现邻居位置 | 否 | 是 | 仅用于生成观测和评估 |
| 未发现邻居波束状态 | 否 | 是 | 仅用于生成事件和评估 |
| 真实拓扑 | 否 | 是 | 仅用于指标计算 |
| 全局代数连通度 | 否 | 是 | 仅用于评估 |

## 8. MVP 简化

第一版仿真采用以下简化：

- 静态或慢变 UAV 位置，用多轮独立种子模拟拓扑变化。
- 同步时隙、无中心调度。
- 理想主瓣覆盖，暂不建模旁瓣。
- 冲突模型先采用同 beam cell 多发冲突。
- ISAC observation 由真实 beam occupancy 加虚警、漏检和角度扰动生成。
- 角度误差用 beam-cell 混淆核建模：真实目标在 `b*`，观测概率扩散到相邻 cells。

后续扩展：

- 异步时隙。
- UAV 移动模型。
- 旁瓣和波束泄漏。
- 更真实的信道和遮挡模型。
